注意事项：如果倒角修改器启用了钳制重叠，同时模型的面存在面积小于倒角长度的面的话，那么倒角修改器只会添加面积等同于小面面积的倒角边（倒角长度等同于最小面的长度；或者说木桶最短的那块板的宽度）。在这种情况下，要么选中并分离面积较小的面作为另一个物体（并调整另一个物体的倒角修改器的倒角）；要么清除有关面的边线的倒角权重，并使用CTRL+B手动倒角这些边线。

# 这个插件有什么用？

我这边在建模的时候不会给中模的锐角添加循环边（支撑环），而是直接使用锐边（并在此基础上添加缝合边），而这样的中模由于缺少支撑环，不能使用表面细分修改器。为了能让中模使用表面细分修改器，需要给结构边线添加支撑环（Edge Loops）

比较方便的方法是使用倒角来给锐角添加支撑环，那么问题来了：
1. 手动按CTRL+B选择所有需要添加支撑环的边，并使用轮廓为1、段数为2的倒角来添加支撑环的操作是不可逆的破坏性修改，而且工作量很大；
2. 使用倒角修改器根据角度自动添加支撑环可能会添加一些不必要的支撑环，这会导致一些预料之外的形变；
3. 使用倒角修改器和倒角权重来添加支撑环需要手动遍历模型的每个锐角上的边，然后添加倒角权重，这样工作量很大（我上边那个截图的例子就是，自己慢慢点的话要点三千多条边）。

为了节约（3）这一步所需要的时间，最快捷的方法是直接将（中模锐角上的）锐边转换为倒角权重，但是吧，Blender没有内置这个功能。

然后我就直接用AI跑了这么一个插件出来：先用DeepSeek生成插件的代码；因为生成的代码不能直接在5.0版本的Blender中运行（会报错），所以又用Gemini 3 Pro 修了一下；结果就是这么一个插件。（谢谢DeepSeek和谷歌）

话说回来，为什么我不在建模过程中添加循环边？因为后期修改很麻烦，添加了循环边的中模（高模）在制作低模的过程中需要手动融并循环边并给锐角添加锐边（以及缝合边），它大概是这么一个过程：
* 制作中模>>在中模的制作过程中添加循环边（高模化）>>按需修改中模的顶点 —— 其中包括添加循环边时中模新增的冗余边线>>复制一份副本>>融并副本物体的所有循环边（结构边线附近的冗余边线）并给结构边线添加锐边和缝合边以制作低模>>烘焙高模到低模。

烘焙高模到低模的问题是，高模的柱状物体的截面平面（从正交顶视图看到的那个圆形面）在使用表面细分修改器的时候会变形，导致高模的圆环截面不能覆盖到低模的圆环截面，在【高模烘焙到低模】时低模的面不能击中高模的面，结果低模圆环面的角落要么是透明的，要么是纯黑色的。例如，一个八边形（甚至四边形）的螺丝在使用表面细分修改器后会变得非常圆（发生形变），作为高模烘焙到低模时，低模会因为不够圆而“漏光”，而漏光反映在（低模的）角落附近的纹理的黑点上。

结果，我这边直接使用高模烘焙基础色（Basecolor）纹理，不在基础色方面上使用【高模烘焙到低模】的工作流程，这样一来圆柱形物体的截面就不会有这样的漏光问题。

另外，如果先制作高模，再将高模转换为中模或者低模的话，低模和高模的UV布局可能不同；如果高模和低模的UV布局不同的话（只在低模上添加锐边和缝合边并展开UV，而非在高模上添加缝合边并展开UV，复制副本做成低模，让低模继承高模的UV布局），用高模烘焙的纹理将不能在低模中使用，只能使用那个有缺陷的【高模烘焙到低模】的工作流程，结果就是出炉的低模成品要么（圆柱体的截面的角落）有黑点（因为漏光），要么最后被迫使用低模烘焙基础色纹理。

那样的话还不如先做中模再转高模，高模使用与中模相同的UV布局并在此基础上烘焙高模的纹理。

于是我这边的建模工作流程大概是这样的：
1. 制作中模；
2. 给中模添加缝合边和锐边并展开UV；
3. 复制副本，将所有锐边转换为倒角权重，用倒角修改器（权重）创建循环边、添加表面细分修改器直接将中模转换成高模；
4. 用高模烘焙纹理。由于高模使用与低模相同的UV布局（这么做应该没什么问题）；
5. 若条件允许，直接使用中模作为游戏引擎的资产。例如，如果中模的三角面数量少于6万5千个三角面，则直接导入到 Source Engine，不做任何优化。

可能是因为我比较菜吧，相对来说先做中模、中模转高模的建模效率应该算是比先做高模再做低模要快不少。
